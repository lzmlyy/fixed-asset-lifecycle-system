# 预警中心规格草案

> 状态：规划中
> 对应 SDD 草案：`docs/sdd/phase-9-warning-center-draft.md`

## 1. 功能定义

基于系统内部数据生成管理提醒，例如长期闲置提醒、维修次数较多提醒、待审批提醒、低净值资产提醒等，提升管理人员处理问题的效率。

## 2. 做什么

- 预警规则（长期闲置、维修频繁、待审批超时、低净值、保修到期等）
- 预警列表展示（按等级、类型、状态筛选）
- 预警等级划分（高、中、低）
- 预警处理状态（待处理、处理中、已处理、已忽略）
- 前端预警中心页面

## 3. 不做什么

- 不做预警的自动通知（邮件、短信）
- 不做预警的自动处理
- 不做预警规则的图形化配置（通过代码或配置文件管理规则）
- 不做预警的定时推送

## 4. 数据模型

新增表（通过 migration SQL 创建）：

| 表名 | 说明 |
|---|---|
| `warning_rule` | 预警规则配置 |
| `warning_record` | 预警记录 |

复用已有表：

| 表名 | 说明 |
|---|---|
| `asset` | 资产数据 |
| `asset_repair_order` | 维修记录 |
| `approval_instance` | 待审批数据 |

## 5. 接口规划

- `GET /api/warning/page` - 预警记录分页查询
- `GET /api/warning/{id}` - 预警详情
- `PUT /api/warning/{id}/status` - 更新预警处理状态
- `POST /api/warning/scan` - 手动触发预警扫描

## 6. 前端页面

- 预警中心列表页（新增）
- 预警详情页（新增）
- 预警处理页（新增）

## 7. 预警规则示例

| 规则名称 | 触发条件 | 等级 |
|---|---|---|
| 长期闲置 | 资产状态为 IDLE 超过 90 天 | 中 |
| 维修频繁 | 同一资产 12 个月内维修次数 ≥ 3 | 中 |
| 待审批超时 | 审批实例创建超过 7 天未处理 | 高 |
| 低净值资产 | 资产净值低于原值的 10% | 低 |
| 保修到期 | 资产保修期已过 | 低 |

## 8. 验收标准

- 可生成多种类型的预警
- 预警列表可按等级、类型、状态筛选
- 预警有明确的等级划分
- 预警可标记处理状态
- 前端可查看预警详情和处理预警
