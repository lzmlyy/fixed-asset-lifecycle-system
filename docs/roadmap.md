# 项目后续路线图

## 1. 当前已完成基础

按当前代码真实情况，以下模块已实现：

- 登录鉴权（JWT + HandlerInterceptor）
- 首页驾驶舱（资产总数、原值、净值、状态统计、分类分布、部门排行、折旧趋势）
- 资产台账（分页查询、多条件筛选、新增、编辑、逻辑删除、详情查看）
- 资产分类（分类树 + 分类列表）
- 入库、领用、调拨、维修、报废五个生命周期单据模块
- 审批流（审批模板、审批实例、审批节点、审批记录）
- 用户管理（用户 CRUD、状态切换、密码重置、角色分配）
- 角色管理（角色 CRUD、角色权限分配）
- 权限管理（34 项权限、`@RequirePermission` AOP 注解、前端 `v-permission` 指令）
- 财务数据网页端展示与模拟同步记录
- AI 辅助分析入口与骨架页面
- 操作日志（资产状态变化自动写入 `asset_operation_log`）

## 2. 后续增强模块

以下模块标记为**规划中**，尚未实现，仅作为后续开发方向。

### 2.1 资产生命周期时间线

**目标：**
在资产详情页展示资产从入库、领用、调拨、维修、报废、审批到操作日志的完整记录。

**价值：**
提升资产流转过程的可视化和可追溯性。

**后续任务：**

- 设计时间线 VO
- 整合生命周期单据
- 整合审批记录
- 整合操作日志
- 前端资产详情页增加时间线区域

### 2.2 盘点任务管理

**目标：**
实现网页端盘点任务、盘点明细、盘点结果和盘点报告。

**价值：**
补齐固定资产管理中的定期核对流程。

**后续任务：**

- 设计盘点任务表
- 设计盘点明细表
- 新建盘点任务页面
- 盘点明细核对页面
- 盘点结果确认
- 盘点报告页面

### 2.2 审批流增强

**目标：**
完善我的待办、我的已办、审批详情、审批记录、审批进度展示和驳回重提。

**价值：**
提升流程管理完整度。

**后续任务：**

- 完善待办列表
- 完善已办列表
- 增加审批进度组件
- 增加驳回重提逻辑
- 完善审批详情展示

### 2.3 AI 辅助分析

**目标：**
基于系统内部数据生成资产状态摘要、异常资产提示、维修/报废辅助建议和管理报告辅助内容。

**价值：**
体现系统智能化能力，但不直接修改业务数据。

**后续任务：**

- 设计 AI 分析数据输入结构
- 设计资产状态摘要
- 设计异常资产提示
- 设计维修/报废辅助建议
- 设计管理报告辅助生成页面

### 2.4 预警中心

**目标：**
基于系统内部数据生成管理提醒，例如长期闲置提醒、维修次数较多提醒、待审批提醒、低净值资产提醒等。

**价值：**
提升管理人员处理问题的效率。

**后续任务：**

- 设计预警规则
- 设计预警列表
- 设计预警等级
- 设计预警处理状态
- 前端增加预警中心页面

## 3. 开发优先级

按以下顺序推进：

1. 审批流增强
2. 盘点任务管理
3. 预警中心
4. AI 辅助分析增强

## 4. 开发原则

- 每个新模块先写 SDD（规格驱动开发文档）
- 每个新模块先明确做什么和不做什么
- 数据库变化必须新增 migration SQL
- 不破坏已稳定主链路（登录、资产台账、生命周期单据、审批流、RBAC 权限）
- README 只写当前真实状态
- 详细规划放 docs
- AI 输出只作为辅助参考，不直接修改业务数据
