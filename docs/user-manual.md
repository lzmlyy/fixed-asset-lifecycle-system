# 固定资产管理系统 用户使用手册

> 本手册面向系统交付后的各类最终用户，按"角色—模块"组织，说明各岗位如何使用系统完成日常资产管理工作。系统采用浏览器访问，登录后按角色权限呈现差异化菜单与首页。

---

## 一、系统登录

### 1.1 访问入口

打开浏览器，访问系统地址（默认 `http://localhost:3000`），进入登录页。登录页左侧展示系统名称与功能简介，右侧为登录表单。

### 1.2 登录方式

1. 在右侧表单输入用户名与密码；
2. 点击「登录」按钮提交；
3. 登录成功后自动跳转至首页驾驶舱。

系统采用 JWT 鉴权：登录成功后服务端签发 Token 并由前端保存，后续所有接口请求自动携带 Token 进行身份校验。Token 过期或失效时需重新登录；长时间未操作也会被要求重新登录。

### 1.3 演示账号

所有演示账号密码统一为 `123456`：

| 用户名 | 姓名 | 角色 | 说明 |
|--------|------|------|------|
| `admin` | 系统默认 | ADMIN | 开发默认账号，拥有全部权限，仅用于开发调试 |
| `system.manager` | 系统管理员 | ADMIN | 系统管理员，全部模块入口 |
| `asset.manager` | 张伟 | ASSET_MANAGER | 资产管理员 |
| `dept.leader` | 李娜 | DEPT_LEADER | 部门负责人 |
| `finance.officer` | 陈敏 | FINANCE | 财务人员 |
| `audit.officer` | 王强 | AUDITOR | 审计人员 |
| `office.staff` | 刘洋 | OFFICE_STAFF | 普通员工 |
| `inventory.clerk` | 赵磊 | INVENTORY_CLERK | 盘点人员 |

### 1.4 登录失败处理

- 用户名或密码错误时页面会提示错误信息；
- 账号被禁用无法登录；
- 无权限访问的页面会跳转至 403 禁止访问页。

## 二、系统布局

登录后进入主界面，整体分为三个区域：

1. **左侧导航菜单**：深蓝色背景，按角色权限动态显示功能入口；
2. **顶部用户栏**：显示当前页面标题与登录用户名，右侧提供「退出登录」按钮；
3. **主内容区**：灰白背景，展示当前功能页面。

不同角色看到的菜单不同，详见第十七章「多角色菜单」。

## 三、首页驾驶舱

首页驾驶舱根据当前用户角色差异化呈现，帮助不同岗位快速获取关注信息。

### 3.1 顶部欢迎区

展示问候语、当前用户姓名、角色标签、所属部门与今日日期。右侧按角色显示关键摘要徽标（如待办事项数、今日操作数、盘点任务数、同步记录数）。

### 3.2 系统管理员（ADMIN）视角

`admin` 与 `system.manager` 登录后看到：

- **总览统计卡片**：资产总数、资产总原值、累计折旧、资产净值；
- **状态统计卡片**：使用中、闲置、维修中、待报废数量；
- **统计图表**：资产分类分布饼图、部门资产金额排行柱状图、月度折旧趋势图（近 12 个月）；
- **盘点任务区**：当前盘点任务与进度；
- **快捷操作区**：全部模块入口（资产台账、盘点任务、我的待办、折旧报表、财务同步、审计追踪等）。

### 3.3 资产管理员（ASSET_MANAGER）视角

- **资产总览统计**：资产总数、原值、累计折旧、净值；状态统计；
- **统计图表**：分类分布、部门排行、月度折旧趋势；
- **盘点任务区**：当前盘点任务列表与完成进度；
- **快捷操作**：资产台账、盘点任务、我的待办、资产领用、维修申请等。

### 3.4 财务人员（FINANCE）视角

- **财务概览**：本月折旧、资产原值、累计折旧、资产净值四项金额指标；
- **月度折旧趋势图**：近 12 个月折旧变化；
- **财务同步记录**：最近同步批次与成功/失败状态；
- **快捷操作**：折旧报表、财务同步、导出报表等。

### 3.5 审计人员（AUDITOR）视角

- **审计概览**：今日操作、资产变更、审批操作、盘点异常四项统计；
- **最新审计日志**：最近操作记录列表（类型、操作、操作人、时间）；
- **快捷操作**：审计追踪、资产台账等。

### 3.6 部门负责人 / 普通员工视角

- **我的待办**：待处理审批事项列表；
- **快捷操作**：资产领用、维修申请、我的待办等。

### 3.7 盘点人员视角

- **盘点任务**：当前任务与进度；
- **快捷操作**：盘点任务、资产台账、审计追踪等。

> 普通员工（OFFICE_STAFF）首页不展示资产总览统计卡片，仅展示待办与快捷操作。

## 四、资产台账

资产台账是系统核心模块，统一维护固定资产基础信息、状态、保管人、部门及存放位置，支持多条件检索与导出。

### 4.1 常用筛选

页面顶部筛选区提供常用查询条件：

- **资产编号**：精确/模糊匹配；
- **资产名称**：模糊匹配；
- **资产分类**：下拉选择；
- **资产状态**：下拉选择（使用中、闲置、维修中、待报废、已报废等）。

点击「查询」执行检索，点击「重置」清空条件。

### 4.2 高级筛选

点击「高级筛选」展开额外条件：

- **所属部门**：下拉选择（支持基础数据下拉）；
- **使用人（保管人）**：下拉选择；
- **存放地点**：下拉选择；
- **购置日期**：日期范围选择。

### 4.3 列表展示

表格展示资产编号、名称、分类、品牌、规格型号、所属部门、使用人、存放地点、原值、净值、状态、购置日期等列。

- **金额列**：原值与净值采用千分位格式、右对齐、等宽数字显示，便于对账阅读；
- **状态列**：以彩色状态标签区分资产状态；
- **操作列**：提供「查看」「更多」按钮，「更多」下拉包含「编辑」「删除」。

### 4.4 资产详情

点击操作列「查看」进入资产详情页，展示资产完整信息：

- 基础信息：资产编号、名称、分类、品牌、规格型号；
- 价值信息：原值、净值、累计折旧、残值率、使用年限、折旧方法；
- 使用信息：购置日期、所属部门、保管人、存放地点；
- 状态信息：彩色状态标签；
- **生命周期时间线**：见第七章。

### 4.5 新增 / 编辑 / 删除

- **新增资产**：点击「新增资产」打开表单，填写资产名称、分类、购置日期、原值、使用年限、残值率、所属部门、使用人等必填项。资产编号由系统自动生成，累计折旧与净值由系统自动计算，新增资产默认状态为「闲置」。
- **编辑资产**：通过「更多 > 编辑」打开表单修改字段，编辑状态变更会记录操作日志，折旧与净值会重新计算。
- **删除资产**：通过「更多 > 删除」二次确认后执行逻辑删除，数据不会真正从数据库移除，删除操作记入操作日志。

### 4.6 导出

点击「导出 Excel」可按当前筛选条件导出资产台账（见第十四章）。

## 五、资产分类

资产分类用于维护固定资产分类体系与默认折旧年限。

### 5.1 查看方式

点击「资产管理 > 资产分类」进入：

- **左侧分类树**：以树形结构展示分类层级；
- **右侧分类列表**：展示分类编码、分类名称、默认折旧年限、备注等。

### 5.2 分类管理

支持分类的新增、编辑、删除（CRUD），分类用于资产录入时的下拉选择与折旧默认参数填充。

## 六、生命周期单据

生命周朢单据覆盖资产从入库到报废的全过程，包含五类单据，每类单据有对应的状态流转。

### 6.1 资产入库（IN）

创建入库单，资产入库后状态变为 `IDLE`（闲置）。入库是资产生命周期的起点，记录资产进入系统的初始信息。

### 6.2 资产领用（RE）

仅 `IDLE` 状态资产可发起领用，领用后状态变为 `IN_USE`（使用中），同时更新使用部门与保管人。

### 6.3 资产调拨（TF）

支持跨部门调拨资产，调拨后更新所属部门、存放地点与保管人。

### 6.4 资产维修（RP）

创建维修单（初始为 `DRAFT` 状态），资产状态变为 `REPAIRING`。维修完成后根据结果更新资产状态：

- 修复：`REPAIRING` → `IN_USE`；
- 建议报废：`REPAIRING` → `WAITING_SCRAP`。

### 6.5 资产报废（SC）

创建报废单，报废后资产状态变为 `SCRAPPED`。支持从 `IDLE` / `IN_USE` / `REPAIRING` / `WAITING_SCRAP` 状态发起报废。

### 6.6 状态流转总览

```
入库 → IDLE（闲置）
IDLE → 领用 → IN_USE（使用中）
IN_USE/IDLE → 调拨（更新部门/地点/保管人）
IN_USE/IDLE → 维修 → REPAIRING（维修中）→ 修复(IN_USE) / 建议报废(WAITING_SCRAP)
IDLE/IN_USE/REPAIRING/WAITING_SCRAP → 报废 → SCRAPPED（已报废）
```

## 七、资产详情时间线

资产详情页底部展示「生命周期时间线」，聚合展示该资产的全部历史事件。

### 7.1 事件类型

时间线聚合以下七类事件，按时间倒序排列：

- **入库（INBOUND）**
- **领用（RECEIVE）**
- **调拨（TRANSFER）**
- **维修（REPAIR）**
- **报废（SCRAP）**
- **审批（APPROVAL）**
- **操作日志（OPERATION_LOG）**

### 7.2 事件信息

每条事件展示：事件标题、事件类型标签、操作人、事件描述、状态变化（变更前 → 变更后）、单据编号、备注、事件时间。

### 7.3 筛选

时间线右上角提供事件类型下拉筛选，可按单一类型查看（如仅查看维修记录、仅查看审批记录）。

## 八、审批中心

审批中心处理生命周期单据的审批流转，分为「我的待办」与「我的已办」。

### 8.1 我的待办

展示当前用户待处理的审批事项，列表包含业务类型、单据 ID、流程名称、当前节点、申请人、提交时间、状态。操作列提供：

- **查看详情**：打开审批详情弹窗，查看完整审批流程与历史节点；
- **通过**：填写审批意见后通过当前节点；
- **驳回**：填写审批意见后驳回。

### 8.2 我的已办

展示当前用户已处理的审批记录，可查看审批结果与审批意见。

### 8.3 审批记录

每次审批操作（提交、通过、驳回）均记录审批人、审批意见、审批时间与状态，形成可追溯的审批链。

---

## 九、盘点任务

盘点任务模块用于发起资产盘点、跟踪进度并记录盘点结果。

### 9.1 创建盘点任务

点击「新建任务」打开弹窗：

- **任务名称**：必填；
- **范围类型**：全部资产 / 按部门 / 按地点；
- 选择「按部门」时需选择部门，选择「按地点」时需选择地点（均使用基础数据下拉）。

### 9.2 任务列表与进度

任务列表展示任务编号、名称、范围类型、范围描述、状态、盘点进度、起止时间。

- **进度条**：以百分比与「已完成数 / 总数」形式直观展示盘点进度，完成任务显示为成功色；
- **状态标签**：进行中（警告色）/ 已完成（成功色）；
- **操作**：查看明细、完成任务（仅进行中任务可操作，完成后不可再修改明细）。

### 9.3 盘点明细与结果录入

点击「查看明细」打开明细弹窗，展示该任务下每项资产的应在地点、实际地点、应在保管人、实际保管人、盘点结果、盘点时间。

点击「编辑」录入盘点结果，盘点结果包含五种类型，并以颜色区分：

- **正常（NORMAL）**：成功色（绿色）；
- **地点不符（LOCATION_MISMATCH）**：警告色（黄色）；
- **保管人不符（KEEPER_MISMATCH）**：警告色（黄色）；
- **丢失（LOST）**：危险色（红色）；
- **账外资产（EXTRA）**：信息色（灰色）。

异常数量在明细列表中通过彩色标签高亮，便于快速识别问题资产。

### 9.4 完成任务

确认所有明细录入完毕后点击「完成任务」，二次确认后任务状态变为已完成，明细将不可再修改。

## 十、折旧报表

折旧报表模块提供资产价值分析与月度折旧趋势展示，是财务与资产管理岗位的核心报表。

### 10.1 核心指标卡片

页面顶部展示四张核心指标卡片：

- **资产原值**：全部资产购置金额合计；
- **资产净值**：原值减去累计折旧；
- **累计折旧**：已计提折旧总额；
- **本月折旧**：当月计提金额。

### 10.2 月度折旧趋势图

展示近 12 个月的折旧趋势：柱状图为各月折旧额，折线为累计折旧与资产净值，用于观察折旧变化趋势。右上角标注最大月折旧金额。

### 10.3 部门 / 分类统计

- **部门资产价值统计**：柱状图对比各部门资产原值与净值；
- **分类资产价值统计**：柱状图对比各分类资产原值与净值；
- 配套数据表格展示资产数量、原值合计、累计折旧、净值合计、月折旧额、折旧率。

### 10.4 明细 Tab

报表提供多个 Tab 切换：

- **月度折旧明细**：按月份查看每项资产的原值、残值率、使用年限、月折旧额、累计折旧、当前净值、状态，可切换报表月份；
- **部门统计**：各部门折旧汇总；
- **分类统计**：各分类折旧汇总；
- **低净值资产**：净值率较低的资产列表，附带减值风险提示预警，建议关注其使用状态与处置计划；
- **接近报废资产**：接近使用年限的资产，展示已使用月数与剩余月数，剩余 ≤6 月以危险色标识。

### 10.5 导出

点击「导出报表」可导出当月折旧报表（含总览汇总行 + 月度明细）。

## 十一、财务同步

财务同步模块记录资产折旧数据与财务系统的同步情况。

### 11.1 总览卡片

展示同步记录总数、最近同步月份、累计同步折旧金额、最近同步状态。最近同步状态以大号标签突出显示（成功为绿色对勾、失败为红色叉号）。

### 11.2 手动同步

选择同步月份后点击「手动同步」可生成一条同步记录。本模块为网页端模拟同步，用于演示资产折旧数据流转，不会调用外部财务系统。

### 11.3 同步记录列表

列表展示批次号、同步月份、资产数量、原值总额、净值总额、累计折旧、本月折旧额、状态、同步时间、操作人。金额列均右对齐、千分位显示，状态以彩色大标签区分成功/失败。点击「详情」可查看单条同步记录详情。

## 十二、预警中心

预警中心集中展示资产价值、使用年限、闲置、维修、盘点和财务同步等多维度风险。

### 12.1 风险等级统计卡片

页面顶部展示四张风险等级卡片：

- **总预警数**；
- **高风险**：危险色，标注「需立即处理」；
- **中风险**：警告色，标注「关注并跟进」；
- **低风险**：信息色，标注「记录备查」。

### 12.2 预警类型卡片

另展示六张类型数量卡片：低净值资产、接近使用年限、长期闲置、维修超期、盘点异常、财务同步异常。

### 12.3 预警列表

列表展示预警类型、等级、标题、描述、资产编号、资产名称、生成时间、处置建议。

- **高风险突出**：高风险行整行以红色背景高亮，中风险以黄色背景高亮；
- **预警类型标签**：以彩色标签区分（低净值、接近使用年限、长期闲置、维修超期、盘点异常、财务同步异常）；
- **等级标签**：高/中/低风险以深色标签带图标显示；
- **预警原因与建议**：描述列说明预警原因，处置建议列给出处理方向；
- 支持按预警类型与等级筛选，可跳转查看关联资产。

## 十三、AI 辅助分析

AI 辅助分析模块由 DeepSeek 大模型基于资产台账、生命周期单据、折旧、盘点、审批与操作日志等系统数据生成管理建议，辅助资产决策。

### 13.1 分析模式

报告顶部以标签标注分析模式：

- **DEEPSEEK（DeepSeek 大模型分析）**：成功调用大模型生成，标签为成功色，顶部蓝色提示「本报告由 DeepSeek 大模型基于系统数据生成，仅供辅助决策」；
- **RULE_FALLBACK（规则引擎分析）**：未配置 API Key 或模型调用失败时自动降级，标签为警告色，顶部橙色横幅说明降级原因（如「未配置 DEEPSEEK_API_KEY」「模型调用超时」等）。

同时展示模型名称与生成时间，支持「重新生成」与「导出报告」。

### 13.2 七分区结构化报告

报告主体按七个分区结构化展示：

1. **资产总体状态摘要**：整体概况，2-4 句话概括资产整体状况；
2. **主要风险分析**：识别的关键风险及处置建议；
3. **财务与折旧分析**：折旧、净值、财务同步维度的洞察；
4. **运营管理建议**：闲置、维修、盘点等运营维度洞察；
5. **审计关注点**：基于审计摘要提示高风险操作；
6. **下一步处置建议**：编号列出的具体处置建议；
7. **管理结论**：总结结论。

### 13.3 辅助明细数据

页面下方提供「辅助明细数据」折叠区（默认收起），展开后包含三张明细卡片：

- **资产状态摘要**：各状态资产数量与价值分布；
- **异常资产提示**：长期闲置、频繁维修、盘点异常资产列表；
- **维修/报废辅助建议**：高频维修与接近报废年限资产建议。

### 13.4 注意事项

- AI 输出仅作为辅助参考，不直接修改业务数据；
- 大模型报告依赖 DEEPSEEK_API_KEY 配置，未配置时自动降级为规则引擎分析，保证可用性。

## 十五、审计日志

审计追踪模块提供全局操作日志与异常追踪，是审计与系统管理岗位的核心工具。

### 15.1 统计卡片

页面顶部展示五张统计卡片：

- **今日操作**：今日操作日志总数；
- **资产变更**：资产相关变更记录数；
- **审批操作**：审批操作记录数；
- **盘点异常**：盘点异常记录数；
- **财务同步**：财务同步记录数。

### 15.2 三级标签

日志列表对每条记录使用三级标签体系：

- **日志类型标签**（深色）：资产操作、审批操作、盘点异常、财务同步；
- **业务类型标签**（浅色）：入库、领用、调拨、维修、报废、盘点、财务同步；
- **来源模块标签**（描边）：记录来源系统模块。

### 15.3 多维筛选

筛选区提供多维度组合查询：

- 日志类型下拉；
- 资产编号输入；
- 资产名称输入；
- 操作人输入；
- 日期范围选择。

支持「查询」「重置」「刷新统计」。

### 15.4 日志详情

点击操作列「查看详情」打开日志详情弹窗，展示日志 ID、类型、资产编号、资产名称、业务类型、业务单据 ID、操作、来源、变更前/后状态、操作人、操作时间、备注等完整字段。点击「查看资产」可跳转至关联资产详情页。

### 15.5 导出

页面右上角提供「导出日志」按钮，按当前筛选条件导出审计日志 Excel。

## 附录：常见问题

1. **忘记密码**：联系系统管理员重置密码；
2. **页面显示无权限**：当前角色未分配该模块权限，联系管理员或切换有权限的账号；
3. **Token 过期**：重新登录即可，系统会自动跳转登录页；
4. **导出无数据**：检查筛选条件是否过严或当前数据范围为空；
5. **AI 报告显示规则引擎模式**：未配置 DEEPSEEK_API_KEY 或大模型调用失败，已自动降级，功能不受影响。
