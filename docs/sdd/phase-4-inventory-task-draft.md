# Phase 4：盘点任务管理阶段草案

> 状态：规划中（草案）
> 开发优先级：3
> 对应规格：`docs/specs/inventory-task-spec.md`

## 1. 阶段目标

实现网页端盘点任务、盘点明细、盘点结果和盘点报告，补齐固定资产管理中的定期核对流程。

## 2. 背景

当前系统已有资产台账、生命周期单据和审批流，但缺少定期盘点核对能力。前端已有盘点任务页面入口和骨架，后端表结构已预留（`inventory_task`、`inventory_detail`），完整业务功能待开发。

## 3. 本阶段做什么

- 盘点任务的创建、编辑、启动、完成
- 盘点明细的生成（按部门/地点/分类范围拉取资产清单）
- 盘点结果录入（正常/盘盈/盘亏/位置异常）
- 盘点报告查看（统计盘盈盘亏数量和明细）
- 盘点异常资产状态标记

## 4. 本阶段不做什么

- 不做盘点任务的自动调度和定时触发
- 不做盘点结果的财务影响计算
- 不做盘点任务审批流（复用已有审批流即可）
- 不接入实物设备能力

## 5. 涉及页面

- 盘点任务列表页（已有骨架，待完善）
- 盘点任务创建/编辑页
- 盘点明细核对页
- 盘点结果确认页
- 盘点报告页

## 6. 涉及后端模块

- `backend/.../inventory/`（已有骨架，待完善）
  - InventoryTaskController
  - InventoryDetailController
  - InventoryReportController
  - 相关 Service 和 Mapper

## 7. 涉及数据表或复用说明

- 复用已有 `inventory_task` 表（init.sql 已建）
- 复用已有 `inventory_detail` 表（init.sql 已建）
- 如需新增字段，通过新增 migration SQL 实现
- 复用 `asset` 表（读取资产清单、更新盘点异常状态）

## 8. 验收标准

- 可创建盘点任务并选择盘点范围（部门/地点/分类）
- 可生成盘点明细清单
- 可录入盘点结果（正常/盘盈/盘亏）
- 可查看盘点报告
- 盘点异常资产状态可标记为 INVENTORY_ABNORMAL
- 所有操作符合 RBAC 权限控制
