# Phase 5：折旧报表与价值分析阶段草案

> 状态：规划中（草案）
> 开发优先级：4
> 对应规格：`docs/specs/depreciation-report-spec.md`

## 1. 阶段目标

基于系统内部资产数据计算和展示资产原值、累计折旧、净值、月度折旧、部门统计和分类统计。

## 2. 背景

当前系统已有资产台账（含原值、残值率、使用年限字段）和折旧记录表（`asset_depreciation`），前端已有折旧报表页面入口。财务模块已有网页端财务数据查看和模拟同步记录。折旧报表的完整业务功能待开发。

## 3. 本阶段做什么

- 月度折旧报表生成
- 按部门统计资产价值（原值、累计折旧、净值）
- 按分类统计资产价值
- 资产净值趋势图展示
- 折旧计算结果展示（平均年限法）

## 4. 本阶段不做什么

- 不做多折旧方法切换（当前只用平均年限法）
- 不做折旧凭证导出到财务软件
- 不做资产减值测试
- 不做税务折旧调整

## 5. 涉及页面

- 折旧报表页（已有入口，待完善）
- 部门资产价值统计页
- 分类资产价值统计页
- 资产净值趋势页

## 6. 涉及后端模块

- `backend/.../depreciation/`（已有预留，待完善）
  - DepreciationReportController
  - DepreciationStatisticsController
  - 相关 Service 和 Mapper
- 复用 `backend/.../finance/` 模块（财务数据查看）

## 7. 涉及数据表或复用说明

- 复用 `asset` 表（原值、残值率、使用年限、入库日期）
- 复用 `asset_depreciation` 表（init.sql 已建）
- 复用 `finance_sync_record` 表（migration-v4-finance.sql 已建）
- 如需新增统计表，通过新增 migration SQL 实现

## 8. 验收标准

- 可按月份生成折旧报表
- 可按部门查看资产价值统计（原值、累计折旧、净值）
- 可按分类查看资产价值统计
- 折旧计算结果与平均年限法公式一致
- 前端可展示趋势图和统计表
- 所有操作符合 RBAC 权限控制

## 9. 后续任务拆分

1. 编写正式 SDD spec 文档
2. 梳理折旧计算规则和边界条件
3. 设计折旧报表接口
4. 设计部门统计接口
5. 设计分类统计接口
6. 实现折旧计算 Service
7. 前端折旧报表页完善
8. 前端统计图表开发（ECharts）
9. 集成测试和验收
