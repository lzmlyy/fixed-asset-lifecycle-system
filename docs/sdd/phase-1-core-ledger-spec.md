# 第一阶段：资产台账核心主链路 · 规格说明

## 1. 背景

国有企业固定资产数量大、价值高、分布广，需要一套系统化的管理工具来替代传统 Excel 管理模式。

第一阶段的目标是搭建系统核心骨架，实现资产台账基本管理能力，为后续生命周期管理奠定基础。

## 2. 目标

- 搭建可运行的前后端分离系统
- 实现登录鉴权
- 实现首页驾驶舱数据概览
- 实现资产台账的增删改查
- 实现资产分类展示
- 建立统一的接口规范和异常处理

## 3. 用户角色

| 角色 | 说明 |
|---|---|
| 系统管理员 | 所有权限 |
| 资产管理员 | 资产台账管理 |
| 财务人员 | 查看资产价值信息 |
| 审计人员 | 只读查看 |

## 4. 功能范围

- 登录：用户名/密码登录，JWT Token
- 首页驾驶舱：资产总数、总原值、累计折旧、净值、状态分布、分类分布、部门排行
- 资产台账：分页查询、条件筛选、新增、编辑、逻辑删除
- 资产详情：展示资产完整信息
- 资产分类：分类树 + 分类列表

## 5. 不做什么

- 生命周期单据流（Phase 2 做）
- 盘点管理（Phase 4 做）
- AI 辅助分析（Phase 7 做）
- 财务数据查看与模拟记录（Phase 5 做）
- 审批流（Phase 3 做）

## 6. 数据规则

- 资产编号：后端自动生成，FA + yyyyMM + 4 位流水号
- 折旧方法：平均年限法，后端自动计算
- 新增资产默认状态：IDLE
- 逻辑删除：deleted 字段标记

## 7. 状态流转

第一阶段资产状态由用户手动编辑，不自动流转。

## 8. 异常规则

- 无效分类：400，资产分类不存在
- 资产不存在：404
- 无权限：401

## 9. 验收标准

- 后端 `mvn -DskipTests package` 通过
- 前端 `npm run build` 通过
- admin/123456 可登录
- 首页驾驶舱显示真实统计数据
- 资产台账可分页查询、新增、编辑、删除
- 无 token 请求返回 401
- 浏览器无白屏、控制台无报错
