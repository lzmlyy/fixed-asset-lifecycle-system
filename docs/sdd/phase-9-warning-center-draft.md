# Phase 9：预警中心阶段草案

> 状态：规划中（草案）
> 开发优先级：5
> 对应规格：`docs/specs/warning-center-spec.md`

## 1. 阶段目标

基于系统内部数据生成管理提醒，例如长期闲置提醒、维修次数较多提醒、待审批提醒、低净值资产提醒等，提升管理人员处理问题的效率。

## 2. 背景

当前系统已有完整的资产台账、生命周期单据、审批流和操作日志，但缺少主动预警能力。管理人员需要手动查询才能发现异常资产和待办事项，效率较低。预警中心通过规则引擎自动生成提醒，帮助管理人员及时发现和处理问题。

## 3. 本阶段做什么

- 设计预警规则（长期闲置、维修频繁、待审批超时、低净值、保修到期等）
- 预警列表展示（按等级、类型、状态筛选）
- 预警等级划分（高、中、低）
- 预警处理状态（待处理、处理中、已处理、已忽略）
- 前端预警中心页面

## 4. 本阶段不做什么

- 不做预警的自动通知（邮件、短信）
- 不做预警的自动处理
- 不做预警规则的图形化配置（通过代码或配置文件管理规则）
- 不做预警的定时推送

## 5. 涉及页面

- 预警中心列表页（新增）
- 预警详情页（新增）
- 预警处理页（新增）

## 6. 涉及后端模块

- `backend/.../warning/`（新增模块）
  - WarningCenterController
  - WarningRuleService（规则引擎）
  - WarningRecordService（预警记录管理）
  - 相关 Mapper

## 7. 涉及数据表或复用说明

- 新增 `warning_rule` 表（预警规则配置）
- 新增 `warning_record` 表（预警记录）
- 复用 `asset` 表（资产数据）
- 复用生命周期单据表（维修、报废记录）
- 复用 `approval_instance` 表（待审批数据）
- 新增 migration SQL 创建预警相关表

## 8. 验收标准

- 可生成多种类型的预警（长期闲置、维修频繁、待审批超时、低净值等）
- 预警列表可按等级、类型、状态筛选
- 预警有明确的等级划分（高、中、低）
- 预警可标记处理状态（待处理、处理中、已处理、已忽略）
- 前端可查看预警详情和处理预警
- 所有操作符合 RBAC 权限控制
