# 固定资产全生命周期管理系统 - 项目最终总结

> 本文档用于课程项目答辩讲解，对系统的建设背景、目标、架构、技术栈、设计与亮点进行系统化说明。

## 一、项目背景

国有企业固定资产种类繁多、数量庞大、流转环节复杂，传统依赖手工台账与 Excel 的管理方式存在明显短板：

- **数据分散易丢失**：资产信息散落在多个部门的多份表格中，版本混乱、难以追溯。
- **流程线下流转**：入库、领用、调拨、维修、报废等环节依靠纸质单据或邮件流转，效率低且易脱节。
- **折旧核算滞后**：财务折旧依赖人工按月计算，工作量大、易出错，难以及时反映资产真实价值。
- **盘点周期长**：实物盘点耗时长、差异处理不及时，账实不符问题突出。
- **风险难发现**：长期闲置、低净值、频繁维修等风险资产缺乏统一视图，决策缺少数据支撑。

在此背景下，本项目面向国企固定资产管理场景，建设一套覆盖资产全生命周期的数字化管理平台，实现"一本账管到底、一张网看全局"。

## 二、建设目标

1. **全流程线上化**：资产从入库、领用、调拨、维修到报废的全生命周期单据闭环，状态可追溯。
2. **财务折旧自动化**：按平均年限法自动核算月度折旧、累计折旧与净值，财务对接有据可依。
3. **盘点作业规范化**：盘点任务下发、明细录入、异常处理、账实核对一体化。
4. **风险预警智能化**：长期闲置、频繁维修、低净值、接近年限等风险自动识别与提示。
5. **AI 辅助决策**：接入 DeepSeek 大模型，基于多维度数据生成自然语言分析报告，辅助管理者决策。
6. **多角色协同**：基于 RBAC 实现 7 个岗位的差异化工作台，做到"各司其职、权限隔离"。

## 三、系统架构

系统采用**前后端分离**架构，浏览器统一访问：

```
┌──────────────┐     HTTPS/JSON     ┌──────────────────┐     JDBC      ┌──────────┐
│   浏览器      │  ←──────────────→  │  Spring Boot 后端 │  ←─────────→  │  MySQL 8 │
│  Vue 3 SPA   │                    │  (REST API + JWT) │               │          │
└──────────────┘                    └────────┬─────────┘               └──────────┘
                                             │  HTTPS
                                             ▼
                                     ┌──────────────────┐
                                     │  DeepSeek 大模型  │
                                     │  (可选，失败兜底)  │
                                     └──────────────────┘
```

- **前端**：Vue 3 单页应用，负责界面渲染、路由守卫、权限指令与图表展示。
- **后端**：Spring Boot 提供 REST API，统一 JWT 鉴权，MyBatis-Plus 持久化。
- **数据层**：MySQL 8 存储全部业务数据，逻辑删除保证数据可追溯。
- **AI 层**：后端按需调用 DeepSeek API，失败时自动回退规则引擎，保证可用性。

## 五、数据库设计概述

数据库名 `fixed_asset_lifecycle_system`，字符集 `utf8mb4`，核心表如下：

| 模块 | 表名 | 说明 |
|---|---|---|
| 资产台账 | `asset` | 资产主表，含编号、原值、净值、状态、保管人等 |
| 资产分类 | `asset_category` | 树形分类，含折旧年限 |
| 生命周期单据 | `asset_inbound_order` / `asset_receive_order` / `asset_transfer_order` / `asset_repair_order` / `asset_scrap_order` | 5 类单据表，记录各类业务流转 |
| 审批 | `approval_instance` / `approval_node` / `approval_flow` / `approval_record` | 审批实例、节点、流程定义与审批记录 |
| 折旧 | `depreciation_record` | 按资产按月记录折旧明细 |
| 盘点 | `inventory_task` / `inventory_record` | 盘点任务与盘点明细 |
| 财务同步 | `finance_sync_record` | 财务同步批次记录 |
| 操作日志 | `asset_operation_log` | 资产操作与状态变更日志 |
| RBAC | `sys_user` / `sys_role` / `sys_permission` / `sys_user_role` / `sys_role_permission` | 用户、角色、权限及其关联表 |

资产表采用 `deleted` 字段实现逻辑删除，MyBatis-Plus 全局配置自动过滤已删除数据。

## 六、核心业务流程

### 1. 资产状态流转

资产状态枚举：`IDLE`（闲置）→ `IN_USE`（使用中）→ `TRANSFERRING`（调拨中）→ `REPAIRING`（维修中）→ `WAITING_SCRAP`（待报废）→ `SCRAPPED`（已报废），另设 `INVENTORY_ABNORMAL`（盘点异常）。每次状态变更均写入 `asset_operation_log`，形成完整轨迹。

### 2. 生命周期单据闭环

入库单生成资产 → 领用单使资产进入使用 → 调拨单变更归属 → 维修单送修并恢复 → 报废单完成资产退出。每类单据独立编号（前缀 + 日期 + 流水号），状态独立流转，审批通过后驱动资产状态变更。

### 3. 审批流

业务单据提交后创建审批实例，按流程节点依次流转；当前审批人可"通过"或"驳回"，通过则推进下一节点，驳回则终止并回退业务状态。审批动作全部记录在 `approval_record`，可追溯。

### 4. 盘点闭环

创建盘点任务（可按部门/地点圈定范围）→ 生成盘点明细 → 盘点人员录入实际地点/保管人 → 系统比对生成盘点结果 → 异常项资产状态置为 `INVENTORY_ABNORMAL` → 复盘处理后恢复，形成"任务—明细—异常—处置"闭环。

## 七、AI 辅助分析设计

AI 辅助分析模块采用**三层架构**，保证智能性与可用性兼顾：

1. **数据准备层**（`collectAiContext`）：从 `AiAnalysisMapper` 只读查询 **16 类数据**，包括资产总览、状态分布、长期闲置、频繁维修、盘点异常、低净值、接近年限、折旧趋势、财务同步、审计日志、高风险资产 Top10 等，组织成结构化上下文。
2. **DeepSeek 生成层**（`LlmClient`）：调用 DeepSeek 大模型（OpenAI 兼容 API），将上下文组装进 Prompt，要求模型以 JSON 输出结构化报告，约束其不可编造数据。
3. **规则兜底层**：当 API 不可用（网络异常、Key 未配置、限流、服务端错误）时，自动切换规则引擎生成报告，返回 `analysisMode=RULE_FALLBACK`，前端以橙色提示。

输出统一为 `AiReportVO`，包含 **7 个分区**：摘要、关键风险、财务洞察、运营洞察、审计关注、建议、结论。

**安全边界**：API Key 仅从环境变量 `DEEPSEEK_API_KEY` 读取，不硬编码、不入库；日志只记录 provider/model/状态码/耗时，不打印 Key 与完整 Prompt；AI 输出不直接修改任何业务数据，仅作辅助参考。

## 八、权限设计

系统基于 **RBAC** 模型实现细粒度权限控制：

- **权限点**：共 **34 项**权限，覆盖各功能模块的查看与操作。
- **岗位角色**：预设 **7 个**岗位角色——系统管理员（ADMIN）、资产管理员（ASSET_MANAGER）、部门负责人（DEPT_LEADER）、财务人员（FINANCE）、审计人员（AUDITOR）、普通员工（OFFICE_STAFF）、盘点人员（INVENTORY_CLERK），每个角色配有对应演示账号。
- **后端控制**：自定义 `@RequirePermission` 注解 + AOP 切面（`PermissionAspect`），方法级权限校验，无权限抛出 `PermissionDeniedException`。
- **前端控制**：路由 `meta.permission` 配合 `router.beforeEach` 守卫；`v-permission` 指令实现按钮级控制；Pinia `authStore` 维护当前用户权限集。
- **角色差异化**：不同角色登录后菜单项与首页 Dashboard 模块按权限差异化呈现，例如财务首页突出折旧与财务概览，审计首页突出审计概览与日志，盘点人员首页突出盘点任务。

## 九、数据展示与报表导出

### 数据可视化

首页驾驶舱与各报表页基于 **ECharts** 提供多类图表：

- **资产分类分布图**：饼图展示各分类资产数量与原值占比。
- **部门资产排行图**：柱状图展示各部门资产原值/净值排行。
- **月度折旧趋势图**：折线图展示近 12 个月月度折旧与累计折旧趋势。

### Excel 报表导出

基于 **Apache POI**（SXSSF 流式）实现 **10 个**导出接口，覆盖全业务场景：

| 序号 | 接口 | 说明 |
|---|---|---|
| 1 | `GET /api/export/assets` | 资产台账 |
| 2 | `GET /api/export/assets/{id}/timeline` | 资产时间线 |
| 3 | `GET /api/export/approval/records` | 审批记录 |
| 4 | `GET /api/export/inventory/tasks` | 盘点任务 |
| 5 | `GET /api/export/inventory/tasks/{taskId}/records` | 盘点明细 |
| 6 | `GET /api/export/depreciation/report` | 折旧报表 |
| 7 | `GET /api/export/finance/sync/records` | 财务同步记录 |
| 8 | `GET /api/export/warnings` | 预警列表 |
| 9 | `GET /api/export/ai/report` | AI 分析报告 |
| 10 | `GET /api/export/audit/logs` | 审计日志 |

前端采用 blob 方式接收响应流并触发浏览器下载，导出文件含标题行、表头冻结与列宽优化。

## 十、系统亮点

1. **AI 大模型接入**：将 DeepSeek 大模型与业务数据深度融合，把结构化数据转化为自然语言分析报告，从"看数据"升级为"读分析"。
2. **角色差异化工作台**：7 个岗位各有专属首页与菜单，做到"千人千面"，贴合国企多岗位协同场景。
3. **完整生命周期闭环**：5 类单据 + 状态机驱动，资产从入库到报废全程线上、可追溯、可审计。
4. **规则引擎兜底保障可用性**：AI 三层架构确保网络或模型异常时系统仍能输出结构化报告，演示与生产均稳定可用。
5. **审计追踪全覆盖**：所有资产操作、审批动作、状态变更均落日志，审计人员可多维筛选与导出，满足合规要求。

## 十一、风险控制

| 风险项 | 控制措施 |
|---|---|
| API Key 安全 | 仅从环境变量 `DEEPSEEK_API_KEY` 注入，`application.yml` 使用 `${DEEPSEEK_API_KEY:}` 占位符，不硬编码、不入库 |
| AI 业务安全 | AI 输出仅作辅助参考，不直接修改任何业务数据，所有业务操作仍需人工通过对应模块完成 |
| 数据安全 | 资产等核心表采用逻辑删除（`deleted` 字段），MyBatis-Plus 全局过滤，数据可追溯不可物理丢失 |
| 操作可追溯 | `asset_operation_log` 全量记录操作人、操作时间、变更前后状态与来源 |
| 权限隔离 | RBAC + `@RequirePermission` + 路由守卫 + `v-permission` 多层校验，越权访问跳转 403 |

## 十二、后续可扩展方向

1. **移动端**：开发移动 App 或小程序，支持扫码盘点、移动审批与现场报修。
2. **资产二维码标签**：为每项资产生成唯一二维码，扫码即可查看资产详情与生命周期，提升实物管理效率。
3. **预警规则配置化**：将闲置阈值、低净值比例、维修次数等参数由代码常量改为可视化配置，业务人员可自助调整。
4. **更多 AI 模型接入**：抽象 LLM 客户端接口，支持切换至其他大模型（如通义千问、文心一言、智谱 GLM），并提供模型对比与择优能力。
5. **资产 IoT 接入**：对接物联网设备，实时采集资产运行状态，支撑更精细的运维分析。

---

**结语**：本系统以国企固定资产全生命周期管理为核心，融合 RBAC 权限、自动化折旧、闭环单据、智能预警与 AI 辅助分析，构建了一套功能完整、体验统一、可演示可扩展的管理平台，具备良好的工程实践价值与课程答辩展示效果。
